# Chatbot Project

## Overview
This project provides an offline AI chatbot, functioning as a "Local AI Lab Terminal." It leverages Retrieval Augmented Generation (RAG) capabilities with the Llama 3.2 language model, powered by Ollama for local inference. The project includes a user-friendly Streamlit web interface and a FastAPI backend designed for flexible integration with external systems, such as WordPress. The chatbot processes user queries by retrieving and synthesizing information from a local knowledge base of product descriptions to deliver contextual and informed responses.

## Features
*   **Offline AI Chatbot:** Operates entirely locally using Ollama and the Llama 3.2 model, ensuring privacy and performance.
*   **Retrieval Augmented Generation (RAG):** Enhances conversational responses by dynamically querying and incorporating data from a dedicated local knowledge base.
*   **Streamlit User Interface:** Provides an intuitive, sleek, and interactive web-based chat terminal experience.
*   **Local Knowledge Base:** Utilizes a plain text file (`data/product_descriptions.txt`) to store and retrieve product-related information.
*   **FastAPI Bridge:** Offers a robust, lightweight API backend for seamless integration with other applications or front-ends.
*   **CORS Enabled:** The FastAPI bridge includes Cross-Origin Resource Sharing (CORS) configured for secure external front-end interactions.
*   **Dynamic Content Aggregation:** The RAG backend is engineered to intelligently scan, aggregate, and tailor responses based on the product catalog.

## Installation

To set up and run the chatbot locally, follow these steps:

1.  **Clone the Repository:**
    ```bash
    git clone https://github.com/your-username/chatbot.git # Replace with actual repository URL
    cd chatbot
    ```

2.  **Set up Python Environment:**
    Ensure you have Python 3.8+ installed. It's recommended to create and activate a virtual environment:
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    ```

3.  **Install Python Dependencies:**
    Install all required Python libraries:
    ```bash
    pip install streamlit fastapi "uvicorn[standard]" python-multipart ollama
    ```

4.  **Install and Configure Ollama:**
    *   Download and install the Ollama server from [ollama.ai](https://ollama.ai/).
    *   Pull the Llama 3.2 model that the application utilizes:
        ```bash
        ollama run llama3.2 
        # Note: Model names can vary. If 'llama3.2' isn't available, check ollama.ai for 'llama3' or similar.
        ```
    *   Ensure the Ollama server is running in the background.

5.  **Prepare Data:**
    Create a `data/` directory in the project root and place your `product_descriptions.txt` file within it.
    ```
    mkdir -p data
    # Ensure data/product_descriptions.txt contains your product information.
    ```

## Usage

You can interact with the chatbot via its Streamlit UI or integrate with its FastAPI bridge.

### Running the Streamlit UI (Local AI Lab Terminal)

This will launch the interactive web-based chat interface:

```bash
streamlit run app.py
```
Open your web browser to the address provided by Streamlit (typically `http://localhost:8501`).

### Running the FastAPI Bridge

This will start the API server, which can be used for programmatic integrations:

```bash
uvicorn local_bridge:app --host 0.0.0.0 --port 8000 --reload
```
The API documentation (Swagger UI) will be accessible at `http://localhost:8000/docs`.

### Interaction
Once the Streamlit UI is running (and Ollama is active with the Llama 3.2 model), simply type your questions into the chat input field to receive responses generated by the RAG system.

## Tech Stack

*   **Programming Language:** Python
*   **Frontend UI:** Streamlit
*   **Backend API:** FastAPI
*   **LLM Runtime:** Ollama (leveraging Llama 3.2)
*   **RAG Core:** Custom Python scripts for data processing and generation
*   **Data Storage:** Plain text files (`.txt`)
*   **Middleware:** CORS (Cross-Origin Resource Sharing)
